How to:

  • Add new entries by filling in the fields of the table.

  • Do not leave out any fields! (Context is optional, but include it for more information)

  • Preferably, do not use other abbreviations inside the explanations to avoid confusing descriptions.

Abbreviations: The abbreviation field shall only include the abbreviation itself, e.g., API and not API Interface or something!

Long Form: The long form field shall only include the long form of the abbreviation it is referring to!

Explanation: The explanation field shall include a comprehensible description! Try to avoid grammar and spelling errors (use tools like LanguageTool etc.) to make the life of your colleagues easier! Some formatting options might not show up correctly in the dictionary extension, so keep it simple!

Context: The context field shall include a reference to the business context where the abbreviation is used in!

API

Application Programming Interface

Eine Schnittstelle, die es Softwareanwendungen ermöglicht, miteinander zu kommunizieren und Informationen auszutauschen.,APIs werden von Entwicklern genutzt, um Funktionen und Daten von externen Anwendungen oder Plattformen in ihre eigenen Anwendungen zu integrieren.

 

SaaS
